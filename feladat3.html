3. feladat

Háromszög szerkeszthetőségének tesztelése

Egy háromszög megszerkeszthető, ha bármely oldalának hossza kisebb a másik két oldal hosszának összegénél.
Tehát a < b + c, vagy b < a + c, illetve c < a + b

(a kép forrása: #kalauzoló)

TDD szoftvertesztelés / Projektmunka 3.feladat:

Döntsük el hogy szerkeszthető-e vagy nem a háromszög, ha szerkeszthető, akkor térjen vissza a Kerület értékével.

Készítsen metódust a következő feladat megoldására a TDD szabályok betartásával! A lépések végrehajtását itt kell dokumentálnia. A dokumentációban a lépések leírása mellett ki kell fejtenie, hogy mit miért és hogyan csinált! Egy projektben kell megvalósítani a feladatok megoldását és a szükséges egyéb dolgokat!

Ismerjük a háromszög 3 oldalának a hosszát. A háromszög szerkeszthető, ha bármelyik két oldal hosszának az összege nagyobb, mint a harmadik oldal hossza.

Dokumentáció lépésről lépésre:

1. A háromszög oldalainak hossz értékei 1-100 közötti egész számok legyenek a feladatban.
2. Ha nem megfelelő szám érkezik akkor -1 értéket adjon vissza a függvény, ha megfelelőek az értékek, akkor a háromszög kerületének értékével térjen vissza.
3. Értékpárok:

  • a, mindhárom érték negatív szám-> -1
  • b, bármelyik érték negatív szám -> -1
  • c, mindhárom érték nagyobb a maximumnál -> -1
  • d, bármelyik érték nagyobb a maximumnál -> -1
  • e, nulla érték -> -1
  • f, bármelyik két oldal hosszának az összege nem nagyobb mint a harmadik oldal hossza ->-1
  • g, helyes értékpárok
4. Tesztesetek megírása a programban
5. Produkciós kód a programban
6. Minden teszt ZÖLD

IntelliJ IDEA programban készült a feladat megoldása.
Itt láthatod a teszteseteket és a produkciós kódot is: 

  

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;
/**
 * @author koviildi
 * @version 1.0
 */
public class ProjectHaromszSzerk {
    /**
     * Lehetséges bemeneti értékek:
     * háromszög oldalai  : 1 – 100
     * Ha hibás értéket adunk át paraméterként, akkor térjen vissza -1 értékkel
     * Kategóriák:
     * - negatív szám előfordulása
     * - nulla érték előfordulása
     * - bármelyik, vagy mindhárom érték nagyobb a maximumnál
     * - 3 jó érték, de nem szerkeszthető
     * - mindhárom jó érték, és szerkeszthető
     */
    @Test
    @DisplayName("Háromszög szerkeszthetőségének tesztelése")
    public void test_haromszogSzerk(){
        assertEquals(-1, szerk(-1,5,8));
        assertEquals(-1, szerk(-5,-7,-6));
        assertEquals(-1, szerk(5,0,7));
        assertEquals(-1, szerk(0,0,0));
        assertEquals(-1, szerk(150,190,800));
        assertEquals(-1, szerk(150,80,90));
        assertEquals(-1, szerk(10,15,80));
        assertEquals(-1, szerk(22,3,4));
        assertEquals(70, szerk(15,25,30));
    }

    /**
     * A háromszög szerkeszthetőségének feltételeit vizsgáló metódus.
     * @param a háromszög egyik oldalának hossza,
     * @param b a háromszög második oldalának hossza,
     * @param c a háromszög harmadik oldalának hossza.
     * @return -1 ha az értékek egyik eleme is az adott értékhatáron kívül esik,
     * vagy nem teljesül a két oldal összegének feltétele,
     * különben a Kerület eredményét adja vissza.
     */
    private int szerk(int a, int b, int c) {
        if (a < 1 || b < 1 || c < 1)
            return -1;
        if (a > 100 || b > 100 || c > 100)
            return -1;
        if (a+b < c || a+c < b || b+c < a)
            return -1;
        else
            return a+b+c;

    }
}
